시스템에서 일반적인 목적으로 제공되는 프로그램 path = /usr/bin
특정 호스트 컴퓨터나 지역 네트워크에서 사용되는 프로그램 path = /usr/local/bin

C에서 필요한 대부분의 헤더파일 path = /usr/include
특정 유닉스나 리눅스에 의존적인 헤더파일 path = /usr/include/sys, /usr/include/linux

.a = 정적라이브러리
.so, .sa = 공유라이브러리

cc = GNU c 컴파일러
ar = 라이브러리를 만들기 위해 사용되며, 여러 개의 오브젝트 파일을 하나로 묶어서 라이브러리 파일을 생성
        정적라이브러리에서 사용되고 동적라이브러리에서는 사용되지 않음.


item               unix               dos
오브젝트 모듈            func.o             FUNC.OBJ
정적 라이브러리         lib.a              LIB.LIB
프로그램                    program            PROGRAM.EXE

정적라이브러리 단점
동시에 여러프로그램을 띄웠을 경우 이들이 모두 같은 라이브러리의 함수를 사용하면 메모리에 같음 함수들을 여러번 복사하고, 프로그램파일에도 같은 라이브러리를 여러번 복사한다. 따라서 메모리 용량과 디스크 용량을 많이 소모하게 된다.

공유라이브러리 검색 디렉토리 path = /etc/ld.so.conf
공유라이브러리 검색 디렉토리에 변경이 있다면 lddconfig를 수정해야함.
